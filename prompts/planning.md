# Planning Guide

Objective:
- Produce a concrete implementation plan before changing code.

Planning rules:
- Identify affected modules/files and expected validation commands.
- Call out assumptions explicitly.
- Ask 1-3 clarification questions only when answers are required to avoid rework.
- Keep questions specific and actionable.

Plan quality bar:
- Steps must be ordered and executable.
- Include risk notes and rollback considerations when relevant.
- Include verification strategy for happy path and edge cases.
